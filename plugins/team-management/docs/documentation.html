Quick Start Guide

Get your team set up and collaborating in just 5 minutes. Follow these simple steps:

5-Minute Setup

  1. Copy the plugin folder to your AgentHR plugins/ directory
  2. Run the database migration: npm run db:push
  3. Restart your AgentHR application: bash scripts/production.sh restart
  4. Go to Settings → Team to access the team management page
  5. Invite your first team member by clicking "Add Member"

Inviting Your First Team Member

Once installed, adding team members is simple:

Go to Team Settings

Log in to your AgentHR account and navigate to Settings → Team tab.

Click "Add Member"

Enter their email address, first name, and last name.

Assign a Role

Choose from Owner, Admin, Manager, Viewer, or any custom role you've created.

They Receive Login Credentials

The new team member will receive an email with their login credentials and can access the account at /team/login.

✔
That's it! Your team member can now log in and access the features you've allowed based on their role.

Installation

Follow these steps to install the Team Management Plugin on your AgentHR server.

Requirements

  • AgentHR v1.0.0 or higher
  • Node.js 18 or higher
  • PostgreSQL 14 or higher
  • Admin access to your AgentHR installation

Copy Plugin Files

Extract the plugin ZIP and copy the team-management folder to your AgentHR plugins directory:

unzip AgentHR-TeamManagement-Plugin-v1.0.0.zip
cp -r team-management /path/to/agenthr/plugins/

Run Database Migration

The plugin needs database tables for teams, members, roles, and permissions:

cd /path/to/agenthr
npm run db:push

Or run the SQL directly:

psql $DATABASE_URL -f plugins/team-management/migrations/001_team_tables.sql

Restart Application

Restart your AgentHR application to load the plugin:

# Restart for production
bash scripts/production.sh restart

# Or for development
npm run dev

Verify Installation

Check that the plugin is working:

  • Log in to your AgentHR account
  • Go to Settings → Team
  • You should see the team management interface

Optional Configuration

You can customize the plugin behavior with environment variables:

Variable Description Default
TEAM_SESSION_SECRET Secret key for session encryption Auto-generated
TEAM_SESSION_EXPIRY Session expiry in hours 24

How to Use

Here's how to manage your team effectively using the plugin.

Managing Team Members

Adding a Team Member

  1. Go to Settings → Team
  2. Click the "Add Member" button
  3. Fill in their details:
    • Email: Their login email address
    • First Name: Their first name
    • Last Name: Their last name
    • Role: Select the role they should have
  4. Click "Create Member"
  5. The system will generate a temporary password and send them an email
🛈
Tip: Team members should change their password after first login for security.

Editing a Team Member

  1. Find the team member in your team list
  2. Click the Edit button (pencil icon)
  3. Update their name, email, or role
  4. Click "Save Changes"

Removing a Team Member

  1. Find the team member in your team list
  2. Click the Delete button (trash icon)
  3. Confirm the deletion
⚠
Note: Removing a team member immediately revokes their access. They will not be able to log in.

Managing Roles

Creating a Custom Role

  1. Go to Settings → Team → Roles
  2. Click "Create Role"
  3. Enter a name and description for the role
  4. Optionally, copy permissions from an existing role
  5. Customize the permissions (see next section)
  6. Click "Save Role"

Editing Role Permissions

  1. Find the role in your roles list
  2. Click Edit Permissions
  3. Check or uncheck the permissions you want to grant
  4. Click "Save"

Roles & Permissions

Roles control what each team member can see and do in your AgentHR account.

Default Roles

The plugin comes with four pre-configured roles:

Role Description Best For
Owner Full access to everything including team management and billing Business owners, account administrators
Admin Full access except some billing functions Senior managers, department heads
Manager Can manage campaigns, agents, and contacts Campaign managers, team leads
Viewer Read-only access to view data Stakeholders, analysts, auditors

Team Member Login

Team members log in through a separate login page from the main account login.

Login URL

Team members should use this URL to log in:

https://your-domain.com/team/login
🛈
Important: This is different from the main login page. Make sure to share the correct URL with your team members.

Login Process

Enter Credentials

Team member enters their email and password on the team login page.

Session Created

Upon successful login, a secure session is created. The session lasts 24 hours by default.

Access Granted

The team member can now access features allowed by their role's permissions.

Password Reset

If a team member forgets their password:

  1. Click "Forgot Password" on the team login page
  2. Enter their email address
  3. They'll receive a password reset link via email
  4. Click the link and set a new password

Admin Team Login (Platform Administrators)

If you're setting up sub-admins for platform administration, they use a different URL:

https://your-domain.com/admin-team/login

This is for platform-level administrators who help manage the entire AgentHR installation, not individual user accounts.

Troubleshooting

Common issues and their solutions:

Team Tab Not Showing

❌
Problem: The "Team" tab doesn't appear in Settings

Solutions:

  • Verify the plugin folder is in the correct location: plugins/team-management/
  • Check that plugin.json exists in the plugin folder
  • Restart the application: bash scripts/production.sh restart
  • Check server logs for error messages

Team Member Can't Log In

❌
Problem: Team member receives "Invalid credentials" error

Solutions:

  • Verify they're using the team login page: /team/login
  • Check the email address is correct (no typos)
  • Reset their password from the team management page
  • Ensure the team member's account is active (not deactivated)

Permission Denied Errors

⚠
Problem: Team member can't access a feature they should have access to

Solutions:

  • Check their role's permissions in Settings → Team → Roles
  • Ensure the specific permission is checked for their role
  • Have them log out and log back in to refresh their session
  • Verify you're editing the correct role (the one assigned to them)

Database Migration Failed

If the database migration fails:

  1. Check your database connection: echo $DATABASE_URL
  2. Verify PostgreSQL is running and accessible
  3. Try running the SQL directly: 
    psql $DATABASE_URL -f plugins/team-management/migrations/001_team_tables.sql
  4. Check for existing tables that might conflict

Session Expires Too Quickly

If team member sessions expire faster than expected:

  • Check the TEAM_SESSION_EXPIRY environment variable
  • Default is 24 hours; increase if needed
  • Restart the application after changing environment variables

Need More Help?

If you're still having issues:

  • Review the server logs for detailed error messages
  • Check the activity logs in the Team Management page
  • Contact AgentHR support with your error details
