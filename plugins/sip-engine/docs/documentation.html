Overview

The SIP Engine Plugin enables customers to use their own SIP trunks for AI-powered voice calls. This "bring your own trunk" capability provides flexibility, cost control, and enterprise integration.

Two Powerful Engines

Feature ElevenLabs SIP OpenAI SIP
Inbound Calls ✔ Yes ✔ Yes
Outbound Calls ✔ Yes ✖ No
Bulk Campaigns ✔ Yes ✖ No
Agent Types Natural, Flow, Incoming Natural (Incoming)
Recordings Available ✔ Server-side via ElevenLabs API ✖ No server-side recordings (transcripts available)
Provisioning Level Per phone number (individual config) Per project (shared webhook/SIP endpoint)
Real-time Streaming Via ElevenLabs WebSocket Via OpenAI Realtime WebSocket (/api/openai-sip/stream/:callId)

Quick Start Guide

Get SIP calling working in under 10 minutes. Choose your engine based on your needs.

ElevenLabs SIP (Inbound + Outbound)

  1. Verify ElevenLabs API Key is configured in Admin Settings (required)
  2. Go to Phone Numbers → SIP Trunks tab in user dashboard
  3. Click "Add SIP Trunk" and select ElevenLabs SIP engine
  4. Select your provider (Twilio, Plivo, etc.) - defaults auto-fill
  5. Enter credentials from your SIP provider account
  6. Import phone numbers and assign AI agents
  7. Configure your provider to route calls to: sip.rtc.elevenlabs.io:5061

OpenAI SIP (Incoming Only)

  1. Get your OpenAI Project ID from platform.openai.com
  2. Configure Admin Settings → SIP Engine → OpenAI SIP Setup
  3. Enter Project ID and save to generate endpoints
  4. Create a webhook in OpenAI Platform with the provided URL
  5. Copy the Webhook Secret and save it in AgentHR
  6. Create SIP Trunk with OpenAI SIP engine in user dashboard
  7. Configure your provider to route calls to: sip:PROJECT_ID@sip.api.openai.com
💡
Which Engine Should I Use?
Use ElevenLabs SIP if you need outbound calling, campaigns, or Flow agents.
Use OpenAI SIP if you only need incoming calls with GPT Realtime.

Installation

Follow these steps to install and configure the SIP Engine Plugin.

Copy Plugin Files

Copy the sip-engine folder to your AgentHR plugins/ directory:

plugins/
  sip-engine/
    index.ts
    plugin.json
    routes/
    services/
    ui/
    docs/
    migrations/

Run Database Migration

The plugin requires database tables for storing SIP trunks, phone numbers, and call logs:

npm run db:push

This creates the following tables:

  • sip_trunks - SIP trunk configurations
  • sip_phone_numbers - Imported phone numbers
  • sip_calls - SIP call logs and transcripts

Build Plugin UI Bundle

Build the frontend components for the admin and user interfaces:

node scripts/build-plugins.js

Restart Application

Restart your AgentHR server to load the plugin:

bash scripts/production.sh restart
# Or for development:
npm run dev

Verify Installation

Check the server logs for successful plugin registration:

[SIP Engine] Plugin registered (v1.0)
[SIP Engine] Endpoints:
  - /api/sip/trunks (user auth)
  - /api/sip/phone-numbers (user auth)
  - /api/admin/sip (admin auth)
  - /api/openai-sip (webhooks)

ElevenLabs SIP Setup

ElevenLabs SIP provides full inbound and outbound calling capabilities using ElevenLabs Conversational AI.

✔
Recommended for Most Users
ElevenLabs SIP supports both inbound and outbound calls, campaign integration, and all agent types.

Prerequisites

  • ElevenLabs account with Conversational AI access
  • ElevenLabs API key configured in AgentHR admin settings
  • SIP trunk provider account (Twilio, Plivo, Telnyx, etc.)
  • Phone numbers from your SIP provider

Step-by-Step Configuration

Configure Your SIP Provider

Set up your SIP trunk in your provider's dashboard:

Twilio: Go to Elastic SIP Trunking → Create Trunk → Configure credentials

Plivo: Go to Voice → SIP Trunk → Create SIP Endpoint

Telnyx: Go to SIP Trunking → Create Connection

Create SIP Trunk in AgentHR

Navigate to Phone Numbers → SIP Trunks tab:

  1. Click Add SIP Trunk
  2. Enter a descriptive name (e.g., "Twilio Production")
  3. Select ElevenLabs SIP as the engine
  4. Select your SIP provider from the dropdown
  5. Enter your SIP credentials (username/password if required)
  6. Click Create Trunk

Import Phone Numbers

With your trunk created, import your phone numbers:

  1. Click Import Number
  2. Select the SIP trunk to use
  3. Enter phone number in E.164 format: +14155551234
  4. Add an optional label for identification
  5. Select an AI agent to handle calls
  6. Click Import

The system automatically provisions the phone number in ElevenLabs.

Configure Inbound Call Routing

In your SIP provider, route incoming calls to ElevenLabs:

# ElevenLabs Inbound SIP URI
TLS (Recommended): sip:sip.rtc.elevenlabs.io:5061;transport=tls
TCP: sip:sip.rtc.elevenlabs.io:5060;transport=tcp
⚠
Important: Your Origination URI transport (TLS/TCP) must match your trunk's Transport setting in AgentHR!

Test Your Setup

Make a test call to your imported phone number:

  1. Call the number from any phone
  2. The call routes through your SIP provider to ElevenLabs
  3. Your AI agent handles the conversation
  4. Check call logs in AgentHR for status and transcript

Re-Provisioning Phone Numbers

If you update your trunk configuration (credentials, transport, etc.), you need to sync these changes to all phone numbers:

  1. Go to SIP Trunks tab
  2. Click the Re-provision All button (server icon) on your trunk
  3. This updates all phone numbers with the new SIP configuration

Outbound Calling

ElevenLabs SIP supports outbound calls from campaigns:

  • Create a campaign and select an agent with an ElevenLabs SIP phone number
  • Upload contacts or add them manually
  • Start the campaign - calls will dial out via your SIP trunk

OpenAI SIP Setup

OpenAI SIP enables incoming calls only using OpenAI's native Realtime SIP integration with GPT-4o.

⚠
Incoming Calls Only
OpenAI SIP does not support outbound calling or campaigns. Use ElevenLabs SIP if you need outbound capabilities.

Prerequisites

  • OpenAI Platform account with Realtime API access
  • OpenAI Project with billing enabled
  • SIP trunk provider account
  • Phone numbers from your SIP provider

Admin Configuration

Get Your OpenAI Project ID

In platform.openai.com:

  1. Go to Settings → Project → General
  2. Find your Project ID (starts with proj_)
  3. Copy this ID

Configure in AgentHR Admin

Go to Admin Panel → Plugins → SIP Engine → OpenAI SIP Setup:

  1. Enter your OpenAI Project ID
  2. Click Save
  3. Copy the generated SIP Endpoint and Webhook URL

Create Webhook in OpenAI Platform

In OpenAI Platform, go to Settings → Webhooks:

  1. Click Add Webhook
  2. Enter the Webhook URL from AgentHR
  3. Select event: realtime.call.incoming
  4. Save and copy the Webhook Secret

Save Webhook Secret

Back in AgentHR Admin → SIP Engine → OpenAI SIP Setup:

  1. Enter the Webhook Secret from OpenAI
  2. Click Save
🔒
Security Warning: Without a webhook secret, incoming requests cannot be verified. Always configure the webhook secret before going to production.

User SIP Trunk Configuration

Once admin setup is complete, users can create OpenAI SIP trunks:

  1. Go to Phone Numbers → SIP Trunks
  2. Click Add SIP Trunk
  3. Select OpenAI SIP as the engine
  4. Select your SIP provider and enter credentials
  5. Import phone numbers and assign agents

Configure Your SIP Provider

Point your SIP trunk to the OpenAI SIP endpoint:

sip:PROJECT_ID@sip.api.openai.com;transport=tls

Replace PROJECT_ID with your actual OpenAI Project ID.

Phone Number Format Matching

The system automatically handles different phone number formats:

  • +14155551234 (with plus)
  • 14155551234 (without plus)
  • 4155551234 (digits only)

All formats are normalized during lookup to ensure calls connect properly.

API Reference

The SIP Engine provides REST APIs for managing trunks and phone numbers programmatically.

SIP Trunks

GET /api/sip/trunks

List all SIP trunks for the authenticated user.

POST /api/sip/trunks

Create a new SIP trunk.

{
  "name": "Production Trunk",
  "engine": "elevenlabs-sip",
  "provider": "twilio",
  "sipHost": "sip.twilio.com",
  "sipPort": 5061,
  "transport": "tls",
  "username": "your-username",
  "password": "your-password"
}
PUT /api/sip/trunks/:id

Update an existing SIP trunk.

DELETE /api/sip/trunks/:id

Delete a SIP trunk and all associated phone numbers.

POST /api/sip/trunks/:id/test

Test the SIP trunk connection.

POST /api/sip/trunks/:id/reprovision-all

Re-provision all phone numbers on this trunk (sync SIP config changes).

SIP Phone Numbers

GET /api/sip/phone-numbers

List all SIP phone numbers for the authenticated user.

POST /api/sip/phone-numbers/import

Import a phone number to a SIP trunk.

{
  "sipTrunkId": "trunk-uuid-here",
  "phoneNumber": "+14155551234",
  "label": "Sales Line",
  "agentId": "agent-uuid-here"
}
POST /api/sip/phone-numbers/:id/assign-agent

Assign or update the AI agent for a phone number.

POST /api/sip/phone-numbers/:id/reprovision

Re-provision a single phone number's SIP configuration.

DELETE /api/sip/phone-numbers/:id

Delete a phone number from the SIP trunk.

Rate Limiting

The phone number import endpoint is rate-limited to prevent abuse:

  • Limit: 10 imports per minute per user
  • Response: HTTP 429 when exceeded

Security Best Practices

Follow these security recommendations to protect your SIP infrastructure.

🔒
Critical Security Requirements
Always configure webhook secrets and use TLS transport in production environments. Failure to do so may expose your system to unauthorized access.

Credential Storage

  • Database Security: SIP trunk credentials are stored in the database. Ensure your PostgreSQL database uses encryption at rest for production deployments.
  • API Protection: Credentials are not exposed in API list responses; only returned during trunk creation/update operations
  • Secure Transmission: All API communications should use HTTPS/TLS in production

Webhook Verification (OpenAI SIP)

OpenAI SIP uses webhook callbacks for incoming call events. Always configure the webhook secret to verify request authenticity:

  1. When you create a webhook in OpenAI Platform, a signing secret is generated
  2. Enter this secret in AgentHR Admin → SIP Engine → OpenAI SIP Setup
  3. All incoming webhook requests are verified using HMAC-SHA256 signature
  4. Requests with invalid or missing signatures are rejected
⚠
Without a webhook secret: Incoming webhook requests cannot be verified, potentially allowing unauthorized parties to trigger fake call events or access your agents.

Transport Security

Transport Port Encryption Recommendation
TLS 5061 Full encryption ✔ Recommended for production
TCP 5060 None Use only when TLS unavailable

IP Allowlisting

For additional security, configure IP allowlisting in your SIP provider:

  • ElevenLabs SIP: Restrict origination to ElevenLabs IP ranges
  • OpenAI SIP: Restrict to OpenAI's documented IP ranges
  • Consult your SIP provider's documentation for IP ACL configuration

API Rate Limiting

The SIP Engine includes built-in rate limiting to prevent abuse:

  • Phone Number Import: 10 imports per minute per user
  • Trunk Operations: Standard API rate limits apply
  • Exceeding limits returns HTTP 429 (Too Many Requests)

Security Checklist

Production Security Checklist

  1. Use TLS transport for all SIP trunks when your provider supports it
  2. Configure webhook secret for OpenAI SIP (required for security)
  3. Use strong passwords for SIP authentication credentials
  4. Enable IP allowlisting in your SIP provider dashboard
  5. Regularly rotate credentials - update passwords periodically
  6. Monitor call logs for suspicious activity
  7. Review user access to SIP features via plan configurations

Troubleshooting

Phone Number Import Fails

  • Verify ElevenLabs API key is configured in admin settings
  • Check API key has Conversational AI permissions
  • Ensure phone number format is E.164 (+ followed by digits)
  • Check if you've exceeded the import rate limit (10/minute)

Inbound Calls Not Connecting

  • Verify SIP trunk credentials are correct
  • Check your SIP provider's termination/routing is configured correctly
  • Ensure transport (TLS/TCP) matches between provider and trunk settings
  • For ElevenLabs: Use sip.rtc.elevenlabs.io (not the old sip.elevenlabs.io)
  • For OpenAI: Verify webhook secret is configured

Outbound Calls Failing (ElevenLabs SIP)

  • Verify phone number has outbound enabled
  • Check SIP trunk credentials
  • Ensure sufficient credits in ElevenLabs account
  • Review call logs for specific error codes

OpenAI Webhook Not Receiving Events

  • Verify webhook URL is publicly accessible (not localhost)
  • Check webhook secret matches between OpenAI and AgentHR
  • Ensure SSL certificate is valid
  • Check OpenAI Platform webhook delivery logs

Agent Not Responding

  • Verify agent is assigned to the phone number
  • Check agent has a valid system prompt configured
  • For OpenAI SIP: Agent must be a "Natural" type agent
  • Review call logs in AgentHR for error details

Audio Quality Issues

  • Use TLS transport when available (more reliable)
  • Check codec compatibility (G.711 recommended)
  • Verify media encryption settings match provider
  • Check network connectivity and firewall rules

Recordings Not Available (OpenAI SIP)

This is expected behavior. OpenAI SIP calls use WebRTC browser-to-browser connections, which means:

  • No server-side recordings are available
  • Transcripts are still available from OpenAI
  • If you need recordings, use ElevenLabs SIP instead

Frequently Asked Questions

Can I use my existing phone numbers?

Yes! The SIP Engine allows you to use phone numbers from your existing SIP provider. Just import them into AgentHR.

Which engine should I choose?

Use ElevenLabs SIP for most use cases - it supports both inbound and outbound calls, campaigns, and all agent types. Use OpenAI SIP only if you specifically want to use GPT Realtime for incoming calls.

Can I use multiple SIP providers?

Yes! You can create multiple SIP trunks with different providers. Each phone number is associated with one trunk.

Are credentials stored securely?

SIP trunk credentials are stored in the database. For production deployments, ensure your PostgreSQL database uses encryption at rest and configure proper access controls.

What happens if I update my trunk credentials?

After updating credentials, use the "Re-provision All" button to sync the changes to all phone numbers on that trunk.

Can I use SIP phone numbers in campaigns?

Yes, but only with ElevenLabs SIP phone numbers. OpenAI SIP only supports incoming calls.

What's the difference between provisioning per trunk vs per number?

ElevenLabs SIP provisions each phone number individually, giving you granular control. OpenAI SIP provisions at the project level - all numbers share the same configuration.

My provider isn't in the list. What do I do?

Select Generic and manually enter your provider's SIP server details (host, port, transport).
